### Hermes 用户整理 OpenAI、xAI、Gemini、DeepSeek 等模型配置

### Hermes 被用于多个飞书机器人同时回答同一问题

向阳乔木继续分享 Hermes 用法：为每个飞书机器人配置不同模型后，把它们拉到一个群里，可以一次性 @ 所有人回答同一个问题，并正在研究机器人之间互相 @ 讨论。这是多模型评审、群内对比回答和轻量多代理协作的一个实验。

关联推文：[向阳乔木](https://x.com/vista8/status/2056174005795926489)

### ORCA Agent IDE 支持移动端、多账号和多 CLI 检测

向阳乔木介绍开源 Agent IDE ORCA，亮点是提供 iOS 与安卓移动客户端，支持多账号切换，例如多个 ChatGPT 订阅；还能显示 Token 消耗与 5 小时重置时间，并检测本机安装的 Claude Code CLI、Codex CLI、Gemini CLI、Hermes、OpenClaw 等工具。它还支持目录和文件拖拽对话，以及内置 Markdown 预览渲染。

关联推文：[向阳乔木](https://x.com/vista8/status/2056271703203033118)

### llama.cpp 跑通 Qwen3.6-27B，4090 上速度约 60 tokens/s

九原客分享 llama.cpp + Qwen3.6-27B + MTP + 4090 跑通的结果，速度约 60 tokens/s；相比 DeepSeek-V4-Pro 常见的 20-30 tokens/s，他认为体感已经很快。当前 Q3 量化可勉强支持约 100K 上下文，但可能 OOM；如果用于 Coding Agent，更推荐 4090 魔改 48GB 或双 24GB 显卡，并需要最新版本 llama.cpp。

关联推文：[九原客](https://x.com/9hills/status/2056271620503716229)

## 观察

今天的开发者信号比中午更清晰：Codex 相关讨论正在围绕稳定性、高频工作流和创作场景展开；国内开发者则把重点放在可落地工具链上，包括 Hermes 多模型群聊、ORCA Agent IDE、本地大模型部署、公众号内容抓取和 Skills 化知识产品。整体趋势是，AI 开发者不再只比较单个模型能力，而是在搭建一整套能持续产出、能跨设备、能接入团队工具的工作系统。
