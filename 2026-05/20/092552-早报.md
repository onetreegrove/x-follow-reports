### Andrej Karpathy 加入 Anthropic，称未来几年是前沿 LLM 形成期

Andrej Karpathy 宣布加入 Anthropic，表示未来几年对前沿 LLM 的发展尤其关键，他很期待回到研发工作。这个动向对开发者社区影响很大，因为 Karpathy 过去在 AI 教育、工程解释和模型直觉传播上有很强影响力；他加入 Anthropic 也进一步强化了 Claude 阵营的研究与产品叙事。

### Claude Managed Agents 推出自托管沙箱和 MCP tunnels

Claude 在 Code with Claude London 上宣布 Claude Managed Agents 新能力：self-hosted sandboxes 进入 public beta，MCP tunnels 进入 research preview。企业可以把 Agent 运行在自己的网络边界内，默认套用内部安全控制；MCP tunnels 则让 Agent 更容易连接组织内部工具与资源。

这说明 Claude 的 Agent 产品路线正在从“会写代码”扩展到“能在企业安全边界内执行工作”。对需要合规、私有依赖、内部服务访问的团队来说，自托管沙箱比单纯云端执行更容易落地。

关联推文：[Claude](https://x.com/claudeai/status/2056645485696315581)

### Google 发布 Gemini 3.5 Flash，主打编码、Agent 任务、速度和成本

Google 相关账号集中介绍 Gemini 3.5 Flash。Logan Kilpatrick 称它是 Google 当前最强模型，面向真实使用场景优化；Demis Hassabis 进一步提到它在 coding 与 agentic tasks 上优于 Gemini 3.1 Pro，速度达到其他前沿模型的 4 倍，在 Antigravity 中可达 800 tokens/sec，并且成本经常低于一半。

这次定位很明确：Google 想用 Flash 把“便宜、快、能干活”的模型打进开发者工作流，而不是只在榜单上展示大模型能力。后续 Pro 版本仍未发布，留给 Google 继续拔高上限。

关联推文：[Logan Kilpatrick](https://x.com/OfficialLoganK/status/2056792266514329914)、[Demis Hassabis](https://x.com/demishassabis/status/2056904067406860545)

### Gemini Omni 面向视频生成与多模态创作，已进入 Gemini、Flow 和 YouTube

Logan Kilpatrick 介绍 Gemini Omni，称它可以从任意输入创建内容，首批重点是视频，类比“用于视频的 Nano Banana”。该模型已经进入 Gemini App、Flow 和 YouTube，API 支持稍后推出。中文社区也观察到 Gemini Omni Flash 已经在 Flow 中放量，但早期效果反馈并不完全稳定。

这条线值得持续关注：Google 正把视频生成、视频编辑、NotebookLM 讲解视频和 YouTube 创作场景放到同一个多模态产品矩阵里。

关联推文：[Logan Kilpatrick](https://x.com/OfficialLoganK/status/2056787874260164628)、[歸藏 1](https://x.com/op7418/status/2056753335261429823)、[歸藏 2](https://x.com/op7418/status/2056758486755844494)、[向阳乔木](https://x.com/vista8/status/2056770987115651418)、[Emad](https://x.com/EMostaque/status/2056802446194737180)

这组更新说明 Google 正试图把“模型 API、Agent 执行环境、应用生成、Workspace 数据和 IDE”串起来，让开发者从 AI Studio 直接进入可执行的 Agent 产品原型。

### LangChain 发布 LangSmith Engine，瞄准 Agent 错误定位到合并修复的闭环

LangChain 表示，很多团队已经会追踪 Agent 并审查输出，但从发现错误到合并修复仍然手工且缓慢。LangSmith Engine 试图缩短这个过程，把 Agent tracing、问题定位和修复流连接起来。LangChain 还连续分享 Engine 的内部机制，以及如何给通用 Agent 设计长周期评测分组。

对真实生产 Agent 来说，评测不再是单一测试集，而是按任务类型、失败模式、运行阶段和长期目标分层管理。LangChain 这批更新把注意力放在了“如何让 Agent 工程持续改进”。

### NVIDIA Vera 开始交付，定位为 Agent 编排和工具调用的 CPU 中枢

歸藏提到 NVIDIA 开始交付自研通用 CPU Vera，重点优化长期高并发和高吞吐场景，适合作为 Agent 编排和工具调用的中枢：模型推理放在 GPU 上，调度、工具调用、IO 和常驻任务压力由 CPU 承担。

这说明 Agent 基础设施正在从“只看 GPU 推理”转向“GPU + CPU + IO + 调度”的整体系统设计。长时间运行的大量 Agent 需要稳定的内存、连接和工具调用吞吐。

## 观察

今天的主线不是单一模型发布，而是“模型 + Agent 环境 + IDE + 搜索/视频/移动入口”的组合竞争。Google 在一天内把 Gemini 3.5 Flash、Omni、Spark、AI Studio、Antigravity 和 Search AI Mode 串起来，意图非常清楚：让模型能力进入开发、搜索、创作和办公的多个入口。

Claude 的自托管沙箱和 MCP tunnels 则把重点放在企业落地：Agent 必须能进入真实内网、真实仓库和真实工具链，同时又不越过安全边界。LangChain 的 Engine 和 evals 讨论补上了另一块：当 Agent 真正进入生产，持续追踪、评测和修复会成为核心工程能力。
